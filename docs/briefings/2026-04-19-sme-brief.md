# LSB — briefing for an external SME (cognitive / quantitative / cultural anthropology)

**Project:** Latent Structure Benchmark (LSB); public surface name **Cognitive Structure Lab**.
**Operator:** Single operator (Mark).
**Prepared:** 2026-04-19.
**Purpose:** Self-contained handoff for an external AI acting as subject-matter expert in cognitive domain analysis, cognitive anthropology, quantitative anthropology, and cultural anthropology. No repo access assumed.

---

## 1. What LSB is — and, precisely, what it claims to measure

LSB applies Cultural Domain Analysis elicitation protocols — free listing, pile sort, pile interview — to large language models and treats the model **as if** it were an informant. The "as if" is deliberate and load-bearing: LSB does *not* claim LLMs have cognition, culture, or experience.

The formal construct LSB measures is:

> the latent categorical structure of a training corpus, as refracted through a model's training and alignment pipeline, surfaced by applying CDA elicitation protocols to the model as if it were an informant.

The project's plain-language term for this is **the corpus lens**. The vocabulary is policed: "worldview," "believes," "thinks (when applied to models)" are forbidden in all generated text (prompts, ledes, dashboard copy, even commit messages). A Reviewer agent enforces this; the substitutions are published in `ARCHITECTURE.md` §1.5.4.

Stated known limitations (surfaced, not buried, on the public methodology page): prompt sensitivity; English-only in v1; corpus opacity for frontier models; alignment confound (pretraining vs RLHF cannot be separated); temperature/sampling variance; and — most importantly for an anthropologist — the fact that the informant metaphor is methodological, not ontological.

## 3. The elicitation protocol, as implemented

Three versioned prompt templates in `packages/cdb_collect/prompts/v1/` (with eight iterated refinements `v1_s1…v1_s8` for the prompt-sensitivity study). Per run, per domain, per model, the runner executes:

1. **Free list** — `free_list.md`: "list every {{domain_seed}} you can think of; do not explain or categorize; aim for ≥ 30." Temperature 0.7. Output parsed (lowercase, strip punctuation, dedupe preserving first-occurrence order).
2. **Pile sort** — `pile_sort.md`: the model receives the truncated salient list **from the same run** and returns JSON `{piles: [[str, ...], ...]}`. Temperature 0.3 (modal categorization). Structured-output mode used where available; robust JSON parser + retry × 3 elsewhere.
3. **Pile interview** — `pile_interview.md`: for each pile, the model names it. This step is what distinguishes LSB from naive textual-similarity benchmarks — the label surfaces the organizing principle directly.

Defaults: N = 5 runs per `(model, domain, step)` tuple. Each run becomes one `InformantRecord` in an append-only JSONL (`data/raw/informants.jsonl`) with full provenance: SHA256 manifest, provider request ID, and the distinction between `model_id` (user alias) and `model_version_returned` (exact API-returned version — critical because providers silently roll snapshots under moving aliases).

## 4. The analysis pipeline

**All analysis is pure deterministic Python.** No LLM calls in `cdb_analyze` — the ban is enforced by a CI static-import check. The lede generator (one call, to Claude) lives in `cdb_publish` specifically to keep the boundary visible.

Pipeline stages (`packages/cdb_analyze/`, all implemented):

```
raw JSONL → parse → aggregate (N runs per model → consensus)
         → cooccurrence (item × item per model)
         → mds (two separate MDS: item × item and model × model)
         → cluster (hierarchical, for dendrogram)
         → consensus (Romney / Weller / Batchelder)
         → drift (Procrustes across versions)
         → bootstrap (B=500 resamples, CIs on everything)
         → gates (G1 / G2 / G3 validation tests)
         → DomainResult → data/results/{domain}/{version}.json
```

## 5. The specific measures

| Measure | Module | What it does |
|---|---|---|
| **Smith's S salience** | `consensus.py:smiths_s` | Individual salience per item per run: `S = (L − R + 1) / L` where L is list length and R is 1-indexed rank. Composite S = mean over N runs (items absent in a run contribute 0). Per Quinlan 2017, Smith 1993, Smith & Borgatti 1997. |
| **Data-driven salience elbow** | `consensus.py:find_salience_elbow` | Maximum-distance-to-chord on the ranked composite-S curve, clamped to `[min_items = 10, max_items = 60]`. Replaces a fixed `truncation_k` with a scree-plot-style cutoff between the high-salience core and the long tail. |
| **Cross-model shared vocabulary** | `consensus.py:compute_cross_model_consensus` | Pools free lists across all models to produce the "common card deck" used for pile sort, so that co-occurrence matrices are commensurable. Methodologically important — see §7 on how this differs from human CDA. |
| **Co-occurrence** | `cooccurrence.py` | For each item pair, fraction of runs in which they appear in the same pile. Symmetric, diagonal = 1.0. |
| **Cross-model similarity** | `cooccurrence.py` | Mantel-style correlation between two models' co-occurrence matrices on the shared item set, rescaled to [0, 1]. |
| **MDS** | `mds.py` | `sklearn.manifold.MDS(n_components=2, dissimilarity='precomputed')`, with `1 − similarity` as dissimilarity. Two separate maps: item × item (within-domain) and model × model (cross-corpus). |
| **Hierarchical clustering** | `cluster.py` | On the model × model distance matrix; produces the dendrogram. |
| **Cultural consensus** | `consensus.py` | Minimum-residual factor analysis on the informant × item agreement matrix. **λ₁ / λ₂ > 3** is the Romney / Weller / Batchelder threshold for a single shared consensus; ratios below 3 indicate sub-cultures — itself a first-class finding. |
| **Pile-count variance** | `consensus.py:compute_pile_count_stats` | Monitors lumper / splitter behavior across runs (mean, std, min, max of pile counts). |
| **Bootstrap uncertainty** | `bootstrap.py` | B = 500 resamples of raw runs; produces 95% confidence ellipses for MDS coordinates (covariance + Procrustes alignment to the reference solution), CIs for every heatmap cell, CIs for the consensus eigenvalue ratio and for Procrustes drift scores. **Display rule: no point estimate without its uncertainty ever ships to the dashboard.** |
| **Drift** | `drift.py` | Procrustes distance between MDS coordinates of two versions of the same model family, after aligning shared item sets. Becomes a longitudinal view in the dashboard once multiple collection dates accumulate. |
| **Prompt sensitivity** | `sensitivity.py` | 8 paraphrased variants per step on two reference models (Claude Opus + the current GPT flagship); contributes the within-model variance for the G1 gate. |

**Phase 4 gate criteria** (binding — nothing downstream ships if any fails):

- **G1 — Stability.** Within-model prompt variance / between-model variance **< 0.5**.
- **G2 — Signal.** Model-to-model similarity matrix distinguishable from random, **permutation test p < 0.01**.
- **G3 — Replication.** Cluster structure reproduces across two independent full pipeline runs, **Rand index ≥ 0.7**.

## 6. Human grounding

A domain may have **zero, one, or many** human baselines; zero is a normal first-class state, not a placeholder. Baselines live in `data/grounding/{domain}/{baseline_id}/` as `source.md` + `items.txt` + `cooccurrence.csv` + optional `pile_sort_raw.csv` + `grounding_ref.json`. The v1 published baseline is **Romney, Boyd, Moore, Batchelder & Brazill (1996) PNAS** for family terms.

The baseline is injected as a virtual informant into the model × model MDS. Item sets will not align exactly — the loader intersects each baseline's items with the LSB salient item set for the domain and flags small intersections. Where researchers contribute raw subject-level pile-sort data, the baseline gets its own bootstrap ellipse; published aggregate-only matrices are rendered without an ellipse and labeled "published aggregate, uncertainty unavailable." Researcher submissions arrive via a GitHub PR template (Phase 6 deliverable) and are gated by a CDA SME review.

## 7. How LSB differs from human-informant CDA — the critical section

This is where the project wants careful SME scrutiny. The methodological import from cognitive anthropology is real, but several assumptions do not cleanly transfer.

1. **"Informants" are API calls to one stochastic process, not distinct humans from a defined population.** N = 5 runs are closer to iid draws from a single posterior than to five cultural informants. LSB uses temperature and prompt variation as the source of variability, and cultural consensus analysis is applied as a *test* of whether those draws converge (λ₁ / λ₂ > 3) rather than being presumed.

2. **The "culture" under study is a text corpus, not a community.** Cultural signal reaches LSB only through the training corpus and only as refracted by pretraining + RLHF + constitutional fine-tuning + any system prompt. The alignment confound is not separable — and LSB treats that as a *feature* for product-level claims and a *limitation* for corpus-level claims.

3. **Card-deck generation is reflexive.** In human CDA, free lists from informants generate the domain vocabulary; LSB does the same, but the "informants" whose free lists generate the deck are *the same class of entities* then being pile-sorted. The project mitigates this by pooling free lists across all models (`compute_cross_model_consensus`) so no single model dictates the deck, but the SME should scrutinize whether this is adequate.

4. **Pile sort is textual, not physical.** The model returns JSON; no manipulation affordance, no pile-reorganization behavior, no hesitation signals. Pile count is a lumper / splitter proxy but a weak one.

5. **The pile-interview step is inverted.** Human CDA often labels piles post-hoc via researcher interpretation or focused interview; LSB asks the model to label its own piles directly. This produces organizing-principle labels as first-class output — informative, but not the same epistemic object.

6. **Romney / Weller / Batchelder CCM assumes a single shared competence.** Applying it across multiple model families inverts the usual use: the eigenvalue ratio is treated as *evidence about* whether training corpora have converged rather than *an assumption that* they have. Sub-cultures (ratio < 3) are a finding, not a failure.

7. **No IRB, no consent, no subject-protection framework** — because there are no human subjects. The ethical axis shifts: the concerns are around how the project talks *about* named commercial products (defamation, mischaracterization), not about subject welfare. The forbidden-vocabulary policy exists in that frame.

8. **Temporal structure.** Cross-cultural / longitudinal human studies have no clean analog; LSB substitutes cross-version drift (Procrustes distance across the same model family over time). This gives a new temporal axis that human CDA does not have, and loses the cross-cultural axis that human CDA does have.

9. **Claims hierarchy is explicit — and the primary axis is model-to-model, not model-to-human.** The framing has been revised post-F1 SME review: human baselines are **contextual reference points**, not the target of measurement. LSB's primary scientific claim is comparative across model architectures and across time; human comparison is secondary. The revised floor/ceiling reading:

    > **Floor claim (no human baseline needed).** Model A and Model B organize this domain differently from each other.
    >
    > **Ceiling claim (human baseline present).** Model A sits closer to the 1996 Romney human consensus than Model B.
    >
    > The ceiling claim is valid and interesting — humans like to compare machines to themselves — but it is not the primary research question. The primary question is cross-architecture comparison. Most v1 domains are ungrounded and ship with floor-level claims only; this is a normal first-class state, not a degraded mode.

10. **"The mismatch is the finding."** The project's position is that importing a methodology designed for cultural informants into a system that encodes culture without experiencing it is itself the research question — not a flaw to apologize for. The SME's scrutiny of exactly where the mismatch breaks the method's assumptions would be the single most valuable review input the project could receive.

## 8. What the SME review is looking for

Specific, actionable feedback is more useful than general assessment. The most valuable inputs would be:

- **Measure validity.** Are Smith's S, Mantel-style matrix correlation, minimum-residual factor analysis with λ₁ / λ₂ > 3, and Procrustes distance being applied in ways consistent with their standard use in the literature? Any known pathologies that would be triggered by the LSB configuration (N = 5 runs per model, ~12 models, ~30–60 items per domain)?
- **Card-deck reflexivity.** Is pooling free lists across models an acceptable mitigation for the fact that the same class of entity generates both the vocabulary and the sorts? If not, what alternative card-deck derivation would the SME recommend?
- **Consensus threshold.** Is λ₁ / λ₂ > 3 still the appropriate Romney / Weller / Batchelder threshold for n = 12 informants (models) on ≤ 60 items, or does the threshold need adjustment for that regime?
- **Gate thresholds.** Are G1 < 0.5, G2 p < 0.01, G3 Rand ≥ 0.7 plausible operational thresholds, or should they be recalibrated given how different "informant" variance is for models vs humans?
- **What is missing.** Any standard measure in CDA or quantitative anthropology that would be cheap to add and would meaningfully strengthen the claims? (Two candidates already being considered: cultural competence scoring per informant; three-way INDSCAL instead of two-way MDS. Both are deferred.)
- **Framing pushback.** Does the "corpus lens" framing hold up under anthropological scrutiny, or does it still carry too much of the ontological baggage it tries to drop?
