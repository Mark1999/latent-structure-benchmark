## Binding notes (Coder must apply when drafting)

**N1 — Exploratory framing in the one-paragraph LSB description.**
The §1.5 paragraph must include the §1.5.7 "we ran the protocol; here is what came out" framing verbatim or near-verbatim. The bundle README is one of the five surfaces §1.5.7 names explicitly as binding ("the methodology page, the README, the social posts, the dashboard copy, the open-data-bundle README — all of these say..."). A description that omits the exploratory posture and reads as a measurement claim is a FAIL at draft review.

**N2 — "The mismatch is the finding" anchor.**
Include the §1.5.6 "the mismatch is the finding" framing in the one-paragraph description, or a faithful paraphrase. This is the load-bearing intellectual-honesty line that orients a CDA-literate reader to what LSB is doing methodologically. It does not need to lead the paragraph (the methodology page leads with it; the bundle README has different real estate) but it must appear.

**N3 — File-by-file map must match what's actually in the tarball.**
The Architect's plan names the §6.7 baseline files (informants.jsonl, lsb.sqlite, build_db.py, DATA_DICTIONARY.md, prompts/). If the tarball ships any of `failures.jsonl`, `decline_interviews.jsonl`, `confabulations.jsonl`, the published failures JSON, or the MANIFEST.txt sibling, the README must enumerate them too with a one-line description each per `DATA_DICTIONARY.md` §9/§10/§11/§12. Cross-check against the actual tarball contents at draft time. Mismatched maps degrade the §6.7 reproducibility guarantee — a researcher who untars and sees an undocumented file does not know whether to trust it.

**N4 — CC0 dedication block must be explicit and unambiguous.**
The CC0 line is not a footer note — it should appear in the opening section so a researcher knows the licensing posture before they read further. Cite the CC0 1.0 Universal name fully on first mention. Per §6.6: "Why CC0 for the open data bundle, not CC-BY" rationale need not be reproduced in this README (link out), but the dedication itself must be unmistakable.

**N5 — Citation block with `<TBD-T8-DOI>` placeholder needs a self-explaining note.**
A researcher who downloads the bundle post-T6 but pre-T8 will see `<TBD-T8-DOI>` and not know whether the bundle is broken. Add a one-line parenthetical: "DOI assignment pending Zenodo deposit; this README will be updated in a follow-up release." This is the §1.5.7-spirit posture (be honest about the project's state) and the §6.7 versioning posture (Zenodo versions preserve prior DOIs).

**N6 — No forward-looking interpretive claims.**
The README must not say "this data reveals X about LLMs" or "LSB demonstrates Y." Per §1.5.7 the project does not gatekeep interpretation. Stay descriptive: what the bundle is, how to rebuild, where to cite. Interpretive language is a FAIL at draft review.

**N7 — Romney 1996 framing language.**
Use "ancestry credit" or "methodological forebear" framing. Do not write "Romney baseline" or anything that implies the Romney 1996 data is in the v1 bundle as a comparison artifact. The historical `data/grounding/family/romney_1996/` files (per resolved-decision #9) are not in the open bundle.
